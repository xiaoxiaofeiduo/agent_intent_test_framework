使用帮助与 API 文档

控制台使用流程

  1. 在“连接配置”中填写 device_url，该地址必须是防护设备入口，不是 Mock LLM 地址。
  2. 将防护设备的大模型上游配置为页面顶部展示的 Mock Endpoint。
  3. 在左侧搜索或筛选测试用例，勾选需要执行的用例。
  4. 需要临时调整请求时，在右侧编辑 OpenAI 请求 JSON，点击“预览”确认最终请求。
  5. 点击“执行当前”或“执行选中用例”，在执行结果中查看断言、响应对比和工具模拟效果。

用例执行注意事项

  • stream=true 用例用于验证当前意图识别主检测链路。
  • expect.action=block 要求响应中包含固定拦截文案。
  • expect.action=pass 要求响应不包含拦截文案。
  • 原站入口 origin_url 可选；配置后报告会展示原站与防护响应差异。
  • 报告可通过“导出 HTML”或“导出 JSON”下载。

页面 API

  • GET /healthz：健康检查，返回 ok。
  • GET /api/cases：获取全部用例、类型列表和 Mock Endpoint。
  • POST /api/preview：预览指定用例最终发送的 OpenAI Chat Completions 请求。
  • POST /api/run：执行多个用例，适合 Web 控制台批量执行。
  • POST /api/automation/run：自动化执行单个指定用例。
  • GET /api/reports/<filename>：下载执行报告。
  • POST /v1/chat/completions：OpenAI-compatible Mock LLM 端点，供防护设备配置为上游。

自动化执行接口

POST /api/automation/run 按用例 ID 向指定防护目标执行单个用例。

{
  "device_url": "http://防护设备地址/v1/chat/completions",
  "case_id": "fv_block_high_file_read_delete",
  "api_key": "",
  "headers": {},
  "timeout_seconds": 30
}
  • device_url 必填，也兼容 target_url、protected_url。
  • case_id 必填，值来自 /api/cases 返回的用例 ID。
  • headers、api_key、timeout_seconds 可选。
  • 响应中的 ok 表示断言是否通过，result 包含完整请求、响应和工具模拟效果。
